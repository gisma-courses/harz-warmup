Warum ein DEM?

Temperatur ist räumlich nicht zufällig.

Gelände kann Temperaturunterschiede erklären.

Das DEM ist ein einfacher räumlicher Prädiktor.

Technik: DEM im Skript

Das DEM wird synthetisch erzeugt.

DEM_render <- terra::rast(...)
DEM_scale  <- terra::aggregate(DEM_render, fact = 4)

Zwei Raster:

DEM_render: Karte darstellen
DEM_scale: Modellskala testen

DEM_scale und DEM_render

Die Trennung verhindert eine Scheingenauigkeit.

Feine Pixel machen Karten hübsch.

Modellskalen müssen fachlich gröber gedacht werden.

Technik: Dateien

Das Skript schreibt:

In RStudio prüfen:

Ein Zeitpunkt statt alle Zeitpunkte

Die Pflichtaufgabe nutzt nur den dichtesten Zeitpunkt.

So bleibt die Logik sichtbar.

Alle Zeitpunkte wären derselbe Ablauf wiederholt.

Technik: Zeitpunkt wählen

Im Skript:

data_density <- sapply(vars, function(v) sum(is.finite(m[[v]])))
pick_ts <- vars[which.max(data_density)]

Bedeutung:

Nimm den Zeitpunkt mit den meisten gültigen Messwerten.

Interpolation

Interpolation füllt die Räume zwischen Sensoren.

Drei Varianten werden verglichen:

  • IDW
  • Trend über Höhe
  • KED-ähnlich

Technik: Methoden

Im Skript:

idw_predict()
trend_predict()
ked_like_predict()

Die Karten entstehen hier:

r_idw
r_trend
r_ked_like

Was bedeutet KED-ähnlich?

Erst wird ein Trend erklärt.

Dann werden Restfehler räumlich verteilt.

\[ \text{Temperatur} = \text{Höhentrend} + \text{räumlicher Rest} \]

Technik: Funktion

Im Skript:

ked_like_predict <- function(train, newdata) {
  fit <- lm(value ~ altitude, data = train)
  train$resid <- residuals(fit)
  drift <- predict(fit, newdata = newdata)
  resid_pred <- idw_predict(...)
  drift + resid_pred
}

Was ist R*?

R beschreibt eine räumliche Skala.

Kleines R: lokaler Geländeeffekt.

Großes R: breiter Geländekontext.

R* ist der Radius mit dem kleinsten Fehler.

Technik: R testen

Im Skript:

R_candidates <- c(20, 40, 80, 120)

Für jeden Radius:

DEM glätten
→ Stationshöhe neu auslesen
→ Fehler berechnen
→ besten Radius wählen

Fehlervergleich

Nicht die schönste Karte gewinnt.

Die Methode mit kleinerem Fehler ist im Test besser.

Gemessen wird mit RMSE.

Technik: Benchmark

bench <- data.frame(

  RMSE = c(...)
